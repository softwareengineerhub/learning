У нас есть Manager (node) - она тоже может запускать контейнеры, однако обычно у нас есть другая нода - Worker.
Manager и Worker могут общаться между собой.

1 Master и 1 Worker - самый простой случай.
Мы хотим запускасть несколько контейнеров одного типа (для горизонтального масштабирования)

Первый мастер - назовем Leader и добавим несколько других Manager




docker swarm создает для нас внутреннюю суть, настраивает необходимую security infrastructure.
Manager nodes - like brains of organization.
Если нам нужен спец микросервис - мы можем сказать docker swarm, что нам нужно минимум 3 инстанса микросервиса.
И менеджеры должны убедится, что такого-то микросервиса работает 3 штуки


Если node или контейнер упадет docker swarm афтоматически reprovision(переделает) it

Manger могут общаться с Worker
Worker могут общаться между собой по средствам Gossip протокола.

Если Worker перестает работать, а мы хотели для масштабируемости иметь скажем 3 ноды - swarm сделает reprovision.
Если Leader перестает работать - тогда происходит election и для worker который остался без Manager зассайнят Manager из работающих
